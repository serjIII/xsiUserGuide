Creating Action Clips in the Mixer

After you create an action source, the next step is to bring it into the animation mixer as an action clip (see Action Sources and Clips for information). Doing this is sometimes referred to as instantiating the action source.

When you create an action clip on a mixer's track, the stored animation drives the corresponding parameters over the range of frames covered by the clip. The action clip takes precedence over any other animation for the model at that frame — see Actions and Other Animation on an Object for more information.

If you selected the Add Source as a Clip in the Mixer option when you created the source, a clip is already created in the mixer.

Once you have instantiated a clip, you cannot reconnect the animation to a different parameter. Instead, delete the clip and instantiate a new one using a different connection-mapping template.

To create an action clip on an animation track in the mixer:

  1. Open the animation mixer in a viewport or floating window (press Alt+0).

  2. Select the model or one of its children and click Update on the mixer's command bar. If this is the first time you have opened the model's track container, two animation tracks are created by default.

  3. If you want to add the clip to a new track, first create a new track by right-clicking any existing track and choosing Add Track Animation (or press Shift+A).

    Clips cannot overlap on the same track, but you can add clips to the same track to play them in sequence. To mix clips, you must place them on separate tracks and overlap them.

  4. Do one of the following:

    • Right-click the location of the animation track where you want the clip to start and choose Insert Source.

      In the explorer that opens, select a stored action from the Sources model folder. This Sources folder gives you access to all action sources stored for any model in the scene.

    • Open an explorer and drag one or more action sources from the Sources model folder for the scene (press O) to an animation track in the mixer.

      You can also drag an action source from the model's Mixer Sources Animation folder. These are copies of the original sources that are stored at the scene level.

      You can also use this technique to copy an action from another model — see Copying Action Sources between Models.

    An action clip is created on the animation track, representing an instance of the action source. You can modify clips independently of each other without affecting the animation stored in the original action source.

    For more information on using clips in the animation mixer, see Working with Clips.

Creating Action Clips from Files

You can also load different types of action files into the mixer to create action clips: .preset, .xsi, or .eani.

Right-click an animation track and choose Load Source From File to load a file, or drag and drop the files using a file browser:

  • To load .preset action files, see Loading Action Source Presets.

  • To load or a .xsi or .eani action file, see Importing Action Sources Files.

Actions and the Animation Icon

When a parameter is driven by an action clip in the mixer, its animation icon in the explorer changes to look like a little plug.

For more information about this and other states of the animation icon related to actions, see The Animation Icon.

TIP:In the schematic view, any node driven by a mixer is identified with an "M" above it.

Selecting Action Clips and Objects

In addition to the regular way of selecting any clip in the mixer (see Selecting Clips), action clips have some other selection options.

To select the clips driving an element

  • Select an element in the scene and choose Clip Select Driving Actions from the mixer's command bar to select the clips that are driving the element.

    TIP:If you want to see only the fcurves inside the clip that is driving an object's animation, select the animated element, choose Clip Select Driving Actions, and then open the animation editor.

To select objects animated by the current clip

  • To make it easy to see what elements are being animated by a clip at any given time, you can select one or more clips and choose Clip Select Driven Objects from the mixer's command bar (or right-click the clip and choose the same command).

    Chain elements are highlighted in white to show that they are being driven by the selected clip (outlined in white).
