Color4 Array Switch

Category: Processing > Switch

Shader Family: Texture

Output: Color

This shader allows you to create an array of color values whose evaluation is based on an integer switch (input). The evaluation is performed as follows:

  • When the input is equal to the index of an Item, the value of that Item is evaluated and the output (result) flows downstream through the render tree.

  • When input is equal to the index of more than one Item, the first Item (in sequential order from the top) to match gets evaluated and output.

  • If input matches none of the Items, then the default parameter is evaluated and output instead.

All of the parameters can be animated or driven by another shader in the render tree. You can also bypass this property editor and create input ports using a render tree workflow exclusively. For more information, see Adding Shader Nodes to the Render Tree.

Name

The name of the shader node displayed in the render tree. Enter any name you like, or leave the default.

input

Accepts an integer value that controls which Item's value gets evaluated.

default

Sets the default color to use when the input does not match the index of any of the Items.

Values

Add

Create an Item group that consists of a color value and an integer index (for matching). You can add as many Item groups as you need for the effect.

Clear

Removes all Item groups from the list.

Item

item

Click the Edit Array Item icon (the " blue socket") to open a pop-up menu with the following commands:

Remove: Deletes the Item group from the list. Once an Item is removed from the list, it no longer influences the effect.

Move Up/Move Down: Promotes or demotes the Item group. The position of an Item in the list indicates its order of evalution.

index

An integer value. When the input matches this index, the value for this Item is output through the render tree.

value

Sets the color values to use when the input matches the index for this Item.
