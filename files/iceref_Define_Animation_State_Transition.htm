Define Animation State Transition

Categories

  • Crowds > Animation

Description

Defines a transition between Animation States.

Parameters

Type

Type of transition:

  • Bi-Directional — the transition applies both ways, from Source State to Target State and vice versa
  • One-Directional — the transition applies in one direction only, from Source State to Target State

Source State

The Animation State from which the transition starts.

Target State

The Animation State at which the transition ends.

Ports

Inputs

In Name

Name of the simulation Point_Cloud. Should be Self, when the node is used in ICE Tree Animation_Blending.

Source Mode

  • Normal — Loop position of an animation state involved in transition is updated during transition from one state to another.
  • Freeze — Position of an animation state will not be updated. This mode is convenient when you need to do transition from static or to static pose.

Target Mode

  • Normal — Loop position of an animation state involved in transition is updated during transition from one state to another.
  • Freeze — Position of an animation state will not be updated. This mode is convenient when you need to do transition from static or to static pose.

Duration

Duration of the transition between animation states.

Outputs

Define

Execute

Out Name

Out name of the simulated Point_Cloud
