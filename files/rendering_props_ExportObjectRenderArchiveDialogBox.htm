Export Object Render Archive Dialog Box

To display:

  1. Select an object.

  2. Choose Render Export Selected Object... from the Render toolbar.

For more information, see Exporting Objects to Object Render Archives for Use as Stand-ins [Rendering].

File

Filename

Specify the path and file name for exporting object render archives (*.mia files).

By default archives are saved to the Render_Archives folder of your current project as defined by the Object Archive Template setting in your Rendering Preferences [Preference Reference].

You can modify the output path by entering a new path and filename directly in the filename textbox or by clicking the browse (...) button to select a different location and file name using a browser window.

Note: The file name you specify should use the [Frame] token (see Tokens and Templates) or Ã”#' (see Setting the Frame Padding) if you are exporting a sequence of frames to object render archives. However, don't worry about this too much because Softimage will try to resolve the path and file name for you.

Sequence

Sequence

When enabled, the selected object is exported to an object render archive for the range of frames in the sequence based on the Start frame, End frame, and frame Step settings.

When disabled, the Sequence options are grayed out and the selected object is exported to an object render archive for the current frame only. This is the default.

Start

Sets the first frame of the sequence to be output.

End

Sets the last frame of the sequence to be output.

Step

Sets the increment between output frames. This allows you to skip frames. For example, if you enter 4, every fourth frame is output.

Export Options

Generate display proxies

When enabled, images representing the contents of the archives are written out for each frame. These images are displayed (like billboards) on the stand-in when the viewport's display mode is set to Textured or Textured Decal.

Display proxies are axes-aligned images saved as Softimage Picture files (*.pic) representing the exported object. A display proxy is created for each of the spatial planes corresponding to the Top, Right, and Front parallel projection windows. For example, the object render archives named object_Archive.[1..2].mia, the following display proxies are created: 

	object_Archive.1.xy.pic
	object_Archive.1.xz.pic
	object_Archive.1.yz.pic
	object_Archive.2.xy.pic
	object_Archive.2.xz.pic
	object_Archive.2.yz.pic

Write all frames to a single archive

Note: Only available for renderers that support a single multi-frame object render archive.

When enabled it does exactly as advertised -- writes all frames to a single object render archive file.

When disabled, each frame is written out to a separate object render archive file.
