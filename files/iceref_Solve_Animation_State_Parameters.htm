Solve Animation State Parameters

Categories

  • Crowds > Animation

Description

This node provides different modes to define the Animation State parameters, according to the input value. Therefore, the weights for the different referenced animations and the playback speed are set.

You need to always use this node in combination with the Set Animation State node parameters.

Parameters

Type

  • Selector — Depending on the input, a blending of the different referenced animations is played back. An input value of 0 means that only the first referenced animation is played back. An input value of 0.3 means the first animation is interpolated with the second one on a ratio of 0.3 to 0.7. Any higher values provide a blending a blending between two animations.

    In this mode, the Speed is always set to 1, so the animations are played back in the original recorded velocity.

  • Movement — The velocity of an actor is expected as input value. According to this value, the playback speed of the animations is adjusted accordingly to stride length of the animations. Also, the blending between the different kind of walking is done accordingly to the velocity of the actor.

State

String value representing the name of the Animation State.

If set to Default, then the solving is done for the Animation State that is defined by the Define Animation State node to which this node is connected to.

Ports

Inputs

In Name

Name of the simulation Point_Cloud

Outputs

State Out

Output string value representing the name of the Animation State

Out Name

Out name of the simulation Point_Cloud

Weights

Array of scalar values representing the weight of each animation used for the Animation State

Speed

Playback speed of the Animation State
