Control Instance Animation

This compound acts as a playback control for the animation of the instanced geometry on the particles.

Plug its Execute output into a Port on the ICETree node.

For more information, see Controlling the Instance's Animation [ICE Particle Simulations].

TIP:This compound uses the ShapeInstanceTime attribute which may cause issues when rendering â€” see Rendering Particle Instances [ICE Particle Simulations] for information and workarounds.

Tasks: Particles/Geometry Instancing

Output Ports: Execute

Animation Start Frame

The first frame (in the scene's timeline) of the instance's animation that you want to use for its animation during the particle simulation.

For example, if the instance's animation goes from frames 1 - 50, you can choose to use only frames 20 - 40 of its animation in the particle simulation.

Animation End Frame

The last frame of the instance's animation that you want to use for its animation during the particle simulation.

Loop Method

The method by which you want to loop the instance's animation after the last frame has been reached:

  • Loop starts playing the animation back from the first frame.

  • Clamp stops the animation at the last frame and holds that frame for the duration of the simulation.

  • No Change simply plays the animation through, which is useful for long animation sequences that may match the simulation length. The Playback Speed Multiplier parameter is useful for this option.

    You don't need to set the Animation Start/End Frame values if you select this option.

Playback Speed Multiplier

Multiplies the playback speed of the instance's animation. A value of 1 is the original speed of the instance's animation.
