Bump Add

| Render Tree Usage

Category: Bump

Shader Family: Texture

Output: Color

This shader is provided for render trees created prior to Softimage version 6.0.1. For new render trees, use Bump Map instead.

This shader lets you use an image or define a color as a bump map. In the render tree, this shader can be connected to either the surface shader or the material node.

Creates a bumpy surface by altering the X, Y, and Z coordinates of the material's surface. A negative value perturbs the surface inward, a value of 0 has no effect on the surface, and a positive value perturbs the surface outward, all prior to illumination calculation. This parameter does not affect the object's geometry.

NOTE:You will need to use the Color2Vector shader to connect this shader to a Bump input in the render tree.

Name

The shader's name. Enter any name you like, or leave the default.

Color Input

Defines an color to use. In the render tree, you can use a texture to control this parameter.

Bump Enabled

Switches on the bump map effect.

Bump Input X, Y, Z

Defines the number of steps the bump map uses when applying the map. This parameter is often used to smooth the bump map or make it more jagged.

Render Tree Usage

This shader is connected to a color (red) input on either the surface input of the material node to create a full bump map, or on a color input on a surface shader, such as Diffuse or Specular. This is done when you wish to apply a bump map to a specific parameter only. This shader can be used on its own (entering a color in the Color Input field), or have an image shader connected to it via the Input parameter.
