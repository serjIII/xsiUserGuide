Load Actor Cycles

Use this compound to get all the animations from an imported actor source in a CrowdFX simulation.

This compound should be connected into Set All Actors Cycles in the Modeling region on the Point_Cloud object of a CrowdFX simulation, before particles are emitted. Use a single Load Actor Cycles compound to get all the animations of one actor, and use separate Load Actor Cycles compound for multiple actor sources. Plug all Load Actor Cycles compounds into the same Set All Actors Cycles compound.

Tasks: Crowds/Initialize

Output Ports: Execute

In Name

The name of the animation proxy cloud that stores one of the animations data of an actor proxy. Use the New port to load additional animations for the same actor.

In default pedestrian simulations, the first, second, and third animations are used for the idle, walk, and run cycles respectively. If you connect the animations in a different order, you must specify the Locomotion Cycle IDs in Initialize Locomotion Data.

For default stadium simulations, the first animation is used.
