Explorer Preferences

| Sorting | Fcurve Editor Preferences

Controls various options in explorer views. You can also set any of these options directly from the explorer menus.

To display: Choose File Preferences, expand the Editors node, and click Explorer.

Explorer

Use Wire Colors

Nodes in the explorer display their wireframe colors.

Show Parameter Values

When enabled, the values of any and all selected parameters are displayed next to the parameter name.

Decompose Branch Selection

Displays the children, grandchildren, and so on of a branch-selected object as top-level nodes instead of as a nested hierarchy. This option is available only when the scope is set to Selection.

Track Selection

The explorer window automatically scrolls up or down to display the nodes of selected elements. If multiple elements are selected, the explorer will only keep the first selected element in view.

Current Scope

The current or last-selected scope of the explorer. This value is read-only and displayed for information only. To change the scope, select an option from he Scope menu of an explorer.

Sorting

Sorting Method for...

Scene Scopes

The sorting method used for objects in the scopes listed under Scene in the explorer's Scopes menu, that is, for Scene Root, Layers, Passes, and so on.

  • None (creation) uses the default order, based on when an element was created or parented.

  • Alphabetical sorts the elements alphabetically. Any numeric suffix is sorted in correct numerical order, so Object2 comes before Object10.

  • Type + Alphabetical sorts the elements by type first, and then alphabetically within each type. The types depend on the scope. For example with the Scene Root scope, the explorer lists all the cameras in alphabetical order, then all the lights, models, referenced models, nulls, chains, curves, polygon meshes, NURBS surfaces, text, point clouds, hair objects, control objects, forces, dynamic constraints, implicits, and geoshaders, each in alphabetical order. If the scope contains only one type of element, this option is equivalent to Alphabetical.

Sources and Clips

The sorting method used for elements in the scopes listed under Sources/Clips in the explorer's Scopes menu, that is, for Sources/Clips, Animation, Audio, and so on.

  • None (creation) uses the default order, based on when an element was created or parented.

  • Alphabetical sorts the elements alphabetically. Any numeric suffix is sorted in correct numerical order, so Elementt2 comes before Element10.

  • Type + Alphabetical sorts the elements by type first, and then alphabetically within each type. The types depend on the scope. If the scope contains only one type of element, this option is equivalent to Alphabetical.

  • Used + Alphabetical sorts the elements into used and unused groups, and then sorts alphabetically within each group. This option is available only with the Sources/Clips scopes.

Other Scopes

The sorting method used for elements in all scopes except those listed under Scene or Sources/Clips in the explorer's Scopes menu.

  • None (creation) uses the default order, based on when an element was created or parented.

  • Alphabetical sorts the elements alphabetically. Any numeric suffix is sorted in correct numerical order, so Elementt2 comes before Element10.

  • Type + Alphabetical sorts the elements by type first, and then alphabetically within each type. The types depend on the scope. If the scope contains only one type of element, this option is equivalent to Alphabetical.

Parameters

Sort parameters according to various criteria:

  • None (creation) uses the default order based on when the parameter was created.

  • Alphabetical sorts the parameters alphabetically. Any numeric suffix is sorted in correct numerical order, so Param2 comes before Param10.

  • Layout uses the order in which the parameters appear in their property editor. If a parameter does not appear in the corresponding property editor, it is not listed.
