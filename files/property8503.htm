Delta Property Editor

Stores any modifications made to a referenced model.

To display: In an explorer, click the Delta node stored below the referenced model.

Target

Target

Specifies the model to which the changes will be applied. The delta can target any referenced model in the scene, not just the model that owns it.

Apply

Controls whether the modifications stored in this delta are applied to the target model whenever the model is updated. You can have multiple deltas for a model and specify which ones are applied.

Record

Controls whether new modifications to the target model are stored in this delta. If this box is checked in multiple deltas, the first delta that is recording the appropriate type of modification (set below) is used. If no delta is recording, a new delta is created for new modifications.

Recorded/Applied Modifications

These modifications will be saved in the delta and reapplied when the referenced model is next updated. If you deactivate an option, any new changes you make are not recorded in this delta. Any pre-existing changes are preserved and reapplied when the model is updated. You can edit or remove items stored in the nodes below the delta.

Static Pos and Other Parameter Values

Saves any changes to unanimated parameter values, including transformations. Note that bone translations, because they are neither IK nor FK, are not recorded by the delta to avoid popping. In addition, changes to to profile curves on pre-existing operators, such as the profile of a Taper or Bulge operator, are not stored and are lost when the model is updated.

Fcurves

Saves changes involving function curves, including, new, modified, or deleted keys.

Expressions

Saves changes any involving expressions, including expressions and linked parameters that have been added, modified, or removed.

Constraints

Saves any constraints that have been added, modified, or removed.

Animation Mixer

Saves changes to the model's animation mixer, such as the placement and weighting of clips on tracks. When this option is on, you can choose the type of Mixer Modifications to save: Animation, Shapes, Audio, and Cache. Be aware that if one of these sub-options is on and the model is re-exported with new sources of the corresponding type, the new sources are not available to the referenced model.

Group Relations

Saves changes to group, layer, and partition memberships.

Model Instances

Saves changes to a model's instances.

External Connections

Saves connections to objects in the scene and other models: envelopes, selective lights, and assignments to materials in the scene library. Note that you cannot change associations of any object in a scene to lights inside a referenced model.

New Properties

Saves any new properties that have been added to any object in the model hierarchy, such as custom parameter sets.

Clusters

Saves any new clusters.

Note that you cannot modify pre-existing clusters by adding or removing components.

Cluster Properties

Saves any new texture UV coordinates, vertex colors, weight maps, and envelope weight properties.

Clusters and cluster properties added to referenced models are designed to work with a single resolution only. You can use multiple external model files as long as the geometries have the same topology, that is, the same number of components and indices in the same order, for example, if they are duplicates. If you switch to a resolution that has a different topology, the cluster properties become unconnected, but they reconnect when you switch back to a resolution with the correct topology.

Note this option does not save paint operators. To save these, use the Operators option below.

Operators

Saves new operators. This includes deformations and scripted operators, as well as paint operators on envelope weights, weight maps, and vertex colors.

Advanced

Persistence

Storage

Specifies how the delta data is stored:

  • Internal deltas are stored in the scene.

  • External (native binary) deltas are stored as separate binary files. If you select this value, specify a File Name for the delta file. Note that external deltas are not automatically loaded and applied when you open a scene.

  • External (custom format) deltas require that you use XSIA (XSI Asset description language) or a similar plug-in. You can install the XSIA plug-in by connecting to the SDK Example workgroup as described in SDK Example Workgroup [Customizing Softimage]. For more information about the XSIA plug-in itself, see XSIA Importer/Exporter [SDK Guide].

Status

Shows whether the delta is currently loaded in the scene. If you open a scene with an external delta, change this value from Offloaded to Loaded and then click Apply Delta.

File Name

The name and path of an external delta file. This is available only when Storage is External.

If Usr is on, the path is displayed as you entered it. If Res is on, the resolved path is displayed. You can type a different path or use the Browse (...) button to change locations. Valid names are displayed in white, invalid names are red, and read-only files are gray.

Apply Delta

Applies an external delta. This is necessary because external deltas are initially offloaded when you open a scene. Change the Status and then click this button.
